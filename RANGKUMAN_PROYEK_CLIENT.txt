RANGKUMAN PROYEK
RAFAY IDP - Intelligent Document Processing untuk Order Logistik
Tanggal dokumen: 8 April 2026

1) RINGKASAN EKSEKUTIF
Proyek RAFAY IDP adalah solusi berbasis AI untuk mengekstrak data order logistik dari chat operasional (khususnya format seperti WhatsApp) menjadi data terstruktur yang siap digunakan tim operasional. Sistem ini mempercepat proses input manual, meningkatkan konsistensi data, dan menyiapkan output dalam format kerja kantor (tabel dan Excel).

2) TUJUAN BISNIS
- Mempercepat proses konversi chat order menjadi data operasional.
- Mengurangi kesalahan input manual pada field penting (driver, nopol, rute, waktu loading, tanggal).
- Menstandarkan format data agar siap dipakai untuk monitoring dan eksekusi.
- Menyediakan jejak data terpusat dengan penyimpanan database dan deduplikasi.

3) CAKUPAN SOLUSI SAAT INI
- Kanal input utama: teks chat/log order.
- Ekstraksi entitas utama: tanggal, waktu loading, jumlah unit, jenis unit, lokasi asal, tujuan, driver, nopol, no HP.
- Normalisasi lanjutan: format jam/tanggal, no HP, rute, nama driver, plat nomor.
- Penanganan data nyata yang kompleks:
  - typo label (contoh: "loksai", "drver", "nopool"),
  - multi-order dalam satu chat,
  - konteks tanggal per blok order,
  - revisi data (rev/update) tanpa menambah order palsu.
- Output operasional:
  - tabel hasil ekstraksi di dashboard,
  - ekspor Excel (All dan Assigned),
  - status unit: ASSIGNED / PARTIAL / UNASSIGNED.

4) ARSITEKTUR TEKNIS (RINGKAS)
- `app.py`:
  - pusat orkestrasi pipeline end-to-end,
  - antarmuka Streamlit,
  - post-processing bisnis,
  - integrasi persistence database.
- `src/inference`:
  - pemuatan model IndoBERT fine-tuned,
  - batch processor untuk split chat dan inferensi.
- `src/data_processing`:
  - converter, cleaner, auto-labeler, augmenter untuk kebutuhan training data.
- `src/training/train_bert.py`:
  - training ulang model NER (IndoBERT) dari dataset internal.
- `db/` + `models/`:
  - ORM SQLAlchemy, inisialisasi schema, deduplikasi upload chat dan row hasil parsing.

5) ALUR PROSES END-TO-END
1. User menempelkan chat order pada aplikasi.
2. Sistem membersihkan dan menormalkan teks (termasuk koreksi label typo).
3. Chat dipisah menjadi blok order yang relevan.
4. Model IndoBERT melakukan ekstraksi entitas.
5. Aturan bisnis melakukan validasi, perapihan, kuota unit, pairing driver/phone, dan handling revisi.
6. Data disusun ke format kantor, dihitung status unit, lalu ditampilkan.
7. Data dapat diekspor ke Excel dan (opsional) disimpan ke PostgreSQL dengan dedup hash.

6) TEKNOLOGI UTAMA
- Bahasa: Python
- UI: Streamlit
- NLP/AI: Hugging Face Transformers (IndoBERT token classification), PyTorch
- Data: Pandas, NumPy
- Database: PostgreSQL + SQLAlchemy
- Utility: python-dotenv, openpyxl/xlsxwriter, pytest

7) KUALITAS & VALIDASI
Pengujian mencakup skenario-skenario penting operasional, termasuk:
- ketahanan ekstraksi waktu loading,
- handling typo label,
- multi-identitas driver dalam satu loading,
- revision handling berdasarkan konteks waktu/lokasi/rute,
- prioritas tanggal slot loading,
- uji data nyata dan uji penyimpanan database.

Catatan: di repository tersedia 8 file test Python utama untuk skenario tersebut, ditambah script uji manual berbasis data contoh nyata.

8) ASET PROYEK YANG SUDAH TERSEDIA
- Model hasil fine-tuning siap pakai (`models/indobert_finetuned/final_model`).
- Dataset chat terproses untuk training.
- Sampel data PDF dan chat untuk eksperimen/pengembangan lanjutan.
- Konfigurasi dependency proyek (`requirements.txt`).

9) NILAI TAMBAH UNTUK CLIENT
- Mempercepat turnaround dari chat operasional ke daftar order siap eksekusi.
- Mengurangi human error pada data kritikal.
- Meningkatkan visibilitas status order (assigned/partial).
- Menjadi fondasi digitalisasi proses dispatch/logistik yang scalable.

10) STATUS IMPLEMENTASI
Solusi sudah berjalan end-to-end untuk use case parsing chat order logistik dan menghasilkan output operasional yang bisa langsung digunakan tim.
